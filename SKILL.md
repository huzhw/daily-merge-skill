# 日报合并 — md → Excel

## 文件路径

```
md 需求记录：C:\Users\Administrator\Desktop\报告-{年}年\日报-{年}-{月}月\日报需求记录-{YYYY}-{MM}-{DD}.md
Excel 日报：  C:\Users\Administrator\Desktop\报告-{年}年\日报-{年}-{月}月\日报表格-胡志伟~~{MM}-{DD}.xlsx
合并脚本：   {技能目录}/scripts/merge.py
```

## Excel 结构（14 列）与 md 映射

| 列 | 标题 | 来源 | 说明 |
|----|------|------|------|
| A | *日期 | 当天 | 同日期合并单元格 |
| B | *项目名称 | md"仓库" | 同项目首行填写，后续省略 |
| C | *序号 | 自动递增 | 全局唯一，永不重复 |
| D | *任务描述 | md"需求概述+涉及模块+备注" | 同仓库多任务合并：N、需求概述\n涉及模块\n备注 |
| E | *完成百分比 | 固定 0% | 留空手动填写 |
| F | *任务创建时间 | 当天 | 录入后勿修改 |
| G | *预计完成时间 | **计算** | 从上个任务 G 列按 H/8 个工作日叠加 |
| H | *预计/实际工时 | md"人工工时" sum | 同仓库多任务工时求和 |
| I | 调整预计完成时间 | — | 留空手动填写 |
| J | 实际完成时间 | — | 留空手动填写 |
| K | AI 工时 | — | 留空手动填写 |
| L | 插队序号 | — | 暂不填 |
| M | 延期/调整原因 | — | 暂不填 |
| N | 备注/说明 | — | 暂不填 |

## G 列计算规则

按 8 小时/工作日依次叠加，跳过周六日：

```
当前累计小时 = 0
当前日期指针 = 今天
for 每个新任务:
    当前累计小时 += H
    while 当前累计小时 > 8:
        日期指针 = 下一个工作日
        当前累计小时 -= 8
    任务.G = 日期指针
```

**第一个任务从当天开始算**。如果昨天已有任务，从昨天最后一个任务的 G 列和剩余小时继续叠加。

## 合并规则

### 规则 1：当月累积，只追加不删除
当月 Excel 是累积快照——每天在前一天基础上追加新完成的行，**不删除任何已有行**。

### 规则 2：同仓库合并为一行
md 中同一仓库的所有已完成任务合并为 Excel 的一行。D 列拼接格式：`N、需求概述\n涉及模块\n备注`，多任务之间空行分隔。H 列（工时）为所有任务工时之和。

### 规则 3：A 列按日期合并单元格
当天新增的所有行，A 列合并为一个单元格，日期值只保留一份。

### 规则 4：B 列同项目省略
同一项目的连续行，只在首行写项目名称，后续行 B 列留空。

### 规则 5：序号全局递增
从全表最大序号 +1 开始编号。

## 执行方式

技能被调用时，AI 判断当前日期后执行对应脚本：

**每月第一天：**
```bash
python "{技能目录}/scripts/new_month.py"
```
从模板起新表，序号从 1 开始。

**其他日期：**
```bash
python "{技能目录}/scripts/daily.py"
```
复制昨天 Excel，追加当天新任务。

如果脚本报错或需要调整，AI 排查后修改对应脚本。

---

## 脚本说明

- `new_month.py` — 月初脚本：读模板 → 写入当天数据，序号从 1 计数
- `daily.py` — 每日脚本：读昨天 Excel → 追加当天数据，序号递增

1. 解析当天 md 文件，提取已完成任务
2. 找昨天 Excel 作为基准（当天已存在则用当天）
3. 读基准 Excel 最后一行数据的 G 列值 + 剩余小时（用于叠加计算）
4. 逐任务计算 G 列
5. 在备注说明行之前插入新行
6. A 列当天所有新行合并单元格
7. B 列同项目连续行省略项目名
8. 保存为当天文件
